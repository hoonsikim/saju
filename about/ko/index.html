사주(四柱)란?

사주(四柱)는 글자 그대로 "네 기둥"입니다. 사람의 출생 연·월·일·시를 천간(天干)과 지지(地支)의 조합으로 읽는 한국 전통입니다. 중국 八字(바지)와 뿌리를 공유하지만, 한국에서 수백 년에 걸쳐 정제된 고유의 해석 체계를 가집니다.

이 사이트는 진짜를 줍니다 — 실제 네 기둥, 일주(日主), 오행 균형, 십신(十神), 지장간, 십이운성, 공망, 십년 대운까지. 서양 운세를 동양 옷 입혀 흉내낸 것이 아닙니다.

무엇을 받나요

  • 무료 기본 풀이 — 네 기둥 + 일주 + 오행 균형 + 짧은 종합 해석. 즉시. 가입 없음. 브라우저에서 생성.
  • $7 심층 풀이 — 위 모든 것 + 지장간·십이운성·공망·십년 대운을 하나의 서사로 엮은 AI 장문 해석. 이메일로 발송.

어떻게 작동하나요

생년월일 데이터는 전적으로 당신의 브라우저에서만 계산됩니다. 서버에 저장하지 않고, 팔지 않고, 공유하지 않습니다. 보관하는 데이터는 결제사가 필요로 하는 주문 기록과, 결제 시 입력한 이메일 주소뿐 — 오직 풀이를 발송하기 위해서입니다.

누가 운영하나요

Saju는 김시훈 (Hoonsi Kim)이 운영합니다. 서울 거주 1인 운영자. 소프트웨어 엔지니어 + 프로덕트 배경. 사이트는 1인 프로젝트로 직접 구축·유지합니다 — 팀 없음, 투자자 없음, 데이터 수확 없음.

결제 및 발송

결제는 Gumroad가 Merchant of Record로 처리합니다 (전 세계 세금·규제 자동 처리). 심층 풀이 발송은 완전 자동화: 주문 webhook → AI 생성 → 이메일 발송 — 보통 1분 이내 도착합니다.

방법론(Methodology)

차트 계산은 결정론적이고 재현 가능합니다. 오픈소스 lunar-javascript 라이브러리(v1.7.7) 기반으로, 한국 정통 사주 관습을 따릅니다:

  • 연주(年柱): 입춘(立春) 기준 (음력 정월이 아님)
  • 월주(月柱): 24절기의 절입(節入) 기준
  • 일주(日柱): 60갑자 cycle, 子時(23:00–01:00)는 다음 날 일주로 처리 (夜子時 = 翌日 학파)
  • 시주(時柱): 24시간 → 12지지 매핑
  • 지장간(支藏干), 십이운성(十二運星), 공망(空亡), 대운(大運), 신살(神煞: 천을귀인·도화·역마): 모두 lunar-javascript EightChar 인터페이스로 계산

진태양시(真太陽時) 경도 보정은 현재 적용되지 않습니다 — 출생 도시 입력은 풀이 맥락용일 뿐입니다. 분 단위 정확도가 필요한 경우(시주 경계 케이스)는 자격을 갖춘 명리상담사를 찾으십시오.

출처 및 참고

해석 어휘는 한국과 중국의 표준 고전에서 가져옵니다:

  • 子平真詮 (자평진전) — 격국(格局)과 용신(用神) 프레임워크
  • 滴天髓 (적천수) — 오행 균형과 일주(日主) 강약
  • 三命通會 (삼명통회) — 십신(十神)과 신살(神煞) 종합 정리
  • 窮通寶鑑 (궁통보감) — 계절별 일주 조후(調候)

이 사이트는 자격을 갖춘 명리상담사를 대체하지 않습니다 — 명리 프레임워크를 전 세계 사용자에게 접근 가능하게 만드는 소프트웨어 해석입니다.

AI 모델 카드

$7 심층 풀이는 다음으로 생성됩니다:

  • 모델: Anthropic Claude Haiku 4.5 (claude-haiku-4-5-20251001)
  • 최대 출력 토큰: 6,144 (≈ 2,500–3,500 단어)
  • 재현성: 같은 생년월일 + 같은 readingType → 비슷한 해석 (LLM은 본질적으로 확률적, 동일 출력 보장 안 됨)
  • 시스템 프롬프트: src/reading-prompt.js (MIT 라이선스로 GitHub 공개)
  • 모델로 전송되는 입력: 생년월일·시각·성별(선택)·도시(선택, 풀이 맥락만)·언어·readingType (general/career/love/wealth). 생년월일은 브라우저에서 계산되며, 결정론적인 차트 구조만 모델로 전송됩니다. 원본 생년월일은 서버에 저장되지 않습니다.
